# AGENTS.md — MemoX v1 contract

Agent contract for this repo. Read before editing. Tool-agnostic (Claude Code,
Codex, Cursor, …). Priority: your tool's system rules → this file → the matching
skill → repo guards/CI.

## What this is

**MemoX v1** — a **local-first** flashcard / spaced-repetition study app.
Flutter owns everything: UI, state, domain logic, and on-device persistence.
**There is no remote server in v1** — "backend" here means the app's local
`domain` + `data` layers. (Account sync / Premium are deferred.)

Stack: **Riverpod Annotation** (state + DI) · **Drift / SQLite** (persistence) ·
**go_router** · **Material 3** · design tokens generated from a design kit.

## Golden rules

1. **Design tokens are GENERATED — never hand-edit `lib/core/theme/mx_*.dart`.**
   They carry a `// GENERATED … DO NOT EDIT` header. To change a value: edit the
   kit CSS, then run `node tool/design/gen_tokens.mjs`. `--check` is the drift gate.
2. **The design kit is the source of truth for visuals** —
   `docs/design/MemoX Design System/` (frozen; synced with Claude Design). If the
   kit looks wrong, fix it there and `/design-sync`; do **not** patch Dart to
   diverge. Only tokens the kit actually *uses* are mirrored (unused are pruned).
3. **No raw visual values in UI** — never `Color(0x..)` / literal px. Use
   `MxColors`/`MxSpacing`/`MxRadius`/`MxTypography`/`MxShadows` + the `MxTheme`
   extension.
4. **Strings come from ARB** (l10n), never hardcoded. Sentence case (kit voice).
5. **Respect the layer contracts** (below). Feature UI must not import `data/`,
   datasources, or `dart:io`; go through providers → use cases.
6. **Verify before claiming done** (see Commands). Don't say "done" if a gate was
   skipped or failed.
7. **Accessible by construction — don't port the kit's JSX a11y shortcuts.** The
   kit uses `div onClick`, class-only `disabled`, and icon ligatures as labels;
   the Flutter port must use proper widgets: `InkWell`/`GestureDetector` +
   `Semantics(button:true)` for taps, real disabled (`onPressed/onChanged: null`),
   a `Semantics` label **from ARB** for every icon-only button, radio semantics
   for selection groups, and ≥ `MxSpacing.minTouchTarget` targets.
8. **v1 is local-only — no cloud/account sync.** Any kit "Cloud sync / Sync
   (alpha)" element renders as **local Backup / Restore** or is omitted; save/load
   errors say local persistence, not cloud/offline sync. `account-sync` deferred.

## Architecture (clean, local-first)

```
lib/
├── app/                      # bootstrap: ProviderScope, MaterialApp.router, env
├── core/
│   ├── constants/ error/ logging/ utils/
│   ├── routes/               # go_router (typed routes, shell + bottom nav)
│   └── theme/                # Tier-0 tokens (GENERATED) + app_theme + mx_theme
├── domain/                   # BE core — pure Dart, NO Flutter/Drift imports
│   ├── entities/  repositories/  usecases/     # incl. the 8-box SRS engine
├── data/                     # BE impl — Drift
│   ├── datasources/local/  models/  repositories/
├── presentation/
│   ├── shared/primitives/    # Mx atoms (Button, Chip, Avatar…)
│   ├── shared/composites/    # Mx composites (Card, Scaffold, AppBar, Nav…)
│   └── features/<feature>/{providers,screens,widgets}
└── l10n/
```

**Layer contracts** (one-way: `Feature → Composite → Primitive → Core`):

- **Foundation** (`core/theme`): context-free, token-only.
- **Primitive**: wraps Material; no business logic, no provider usage, no feature
  imports.
- **Composite**: composes primitives; feature-independent; no provider usage.
- **Feature**: screens render + dispatch intent; `@riverpod` providers own
  mutation and call domain use cases. No `data/`/`dart:io` imports in UI.
- **Domain**: pure Dart, immutable, `Result`-returning, deterministic (inject a
  clock — no wall-clock in logic). **`domain/repositories/` interfaces are the
  frozen FE↔BE contract.**
- **Data**: Drift row models stay separate from domain entities; map at the
  boundary. Wire repositories/use cases through `@riverpod` providers.

## Build workflow

The plan is **[`docs/project-management/wbs.md`](docs/project-management/wbs.md)**.
Every task has a self-contained loop prompt under
**[`docs/agent/build/`](docs/agent/build/README.md)** (generated by
`tool/design/gen_task_prompts.mjs`). Run the build with `/loop` over that queue —
it picks the next `[ ]` task by phase + deps.

Order: **I** (infra) unblocks all → **T** (theme) + **DM** (domain contracts)
open two parallel tracks → **FE**: P primitives → K composites → S screens ·
**BE**: DM domain → DT Drift data. Screens use in-memory fakes until DT.5 wires
real repos. **S.01 dashboard is the pilot** — build, review, then fan out.

## Commands

**The verify gate is a single command: `node tool/verify/run.mjs`** (created in
WBS **I.0**). Tasks call it — not the raw commands. Only I.0 itself may bootstrap
with raw commands (it's building the runner).

```bash
# the one gate every task runs before commit
node tool/verify/run.mjs           # full: codegen freshness + gen_tokens --check + analyze + test
node tool/verify/run.mjs --quick   # analyze + test only
node tool/verify/run.mjs --docs    # doc/spec freshness + gen_tokens --check only

# regenerate design tokens after a kit change
node tool/design/gen_tokens.mjs

# regenerate the WBS loop prompts after editing the registry
node tool/design/gen_task_prompts.mjs
```

Until I.0 lands, `tool/verify/run.mjs` does not exist yet; the raw equivalent is
`dart run build_runner build --delete-conflicting-outputs && node tool/design/gen_tokens.mjs --check && dart analyze lib test && flutter test`.

**Definition of done** per task: built at the mapped path (layer-correct) ·
analyzes clean · tested at the right level (domain = pure unit · data = Drift
integration · primitives/composites = widget+golden light+dark · screens =
provider-state widget tests + golden vs `shots/*.png`) · parity/correctness ·
`§Ledger` row in the WBS · all gates green.

## Design-sync (pull the kit)

The kit is pulled/pushed via the official `/design-sync` skill. It runs headless:

```bash
MSYS_NO_PATHCONV=1 claude -p "/design-sync" --dangerously-skip-permissions --max-turns 40
```

`MSYS_NO_PATHCONV=1` is **required** on Git Bash / Windows (else the `/design-sync`
arg is mangled into a path). After a pull, run `node tool/design/gen_tokens.mjs`
and commit. Notes: `.design-sync/NOTES.md`.

## Git conventions

- Branch off `main` before changes (`build/<taskid>` for WBS tasks); PR → merge.
- End commit messages with the `Co-Authored-By:` trailer your tool uses.
- **Design-sync triggers (both PUSH kit → Claude Design):**
  - `pre-push` — on `git push` whose range changes `docs/design/MemoX Design
    System/`.
  - `post-merge` — after `main` receives kit changes (PRs merge server-side +
    agent pushes bypass pre-push, so this closes the loop; compares against
    `lastSyncedCommit`, record mode).
  Both honor `MEMOX_SKIP_DESIGN_SYNC=1` and need a design-authorized `claude` CLI.
- From an **agent session without a design-authorized interactive terminal**,
  prefix **both** `git push` **and** `git pull` (on main) with
  `MEMOX_SKIP_DESIGN_SYNC=1 …` — else the nested `claude` hangs. Not `--no-verify`.
- Generated files (`lib/core/theme/mx_*.dart`, `tool/design/*.mjs`) are pinned to
  LF via `.gitattributes`.
