# WBS вҖ” MemoX v1 (local-first Flutter app)

Full Work Breakdown Structure for MemoX v1: a **local-first** flashcard / SRS
app. Flutter owns everything вҖ” UI, state, domain logic, and local persistence.
There is **no remote server in v1** ("BE" here = the app's on-device domain +
data layers). Built so the **FE track** (design-system вҶ’ components вҶ’ screens)
and the **BE track** (domain вҶ’ SRS engine вҶ’ Drift data) proceed **in parallel**
once shared foundations land.

Confirmed decisions (v1):
8-box single-direction SRS В· 20 new cards/day В· nested decks В· search term+meaning
В· soft-dup В· goal + streak В· Premium/account-sync **deferred**. Stack: **Riverpod
Annotation** (state + DI) В· **Drift/SQLite** (persistence) В· **go_router** В·
Material 3 В· design tokens already generated (Tier 0).

> **Loop execution.** Every task has a self-contained loop prompt under
> [`docs/agent/build/`](../agent/build/README.md) (generated by
> `tool/design/gen_task_prompts.mjs`). Run with `/loop`; the queue picks the next
> pending task by phase + deps.

**Layer contracts** (enforced): foundation is context-free/token-only В·
primitives wrap Material, no business logic / provider / feature imports В·
composites compose primitives, feature-independent В· feature UI may own provider
wiring but must not import repositories/datasources/`dart:io` В· dependency
direction one-way `Feature вҶ’ Composite вҶ’ Primitive вҶ’ Core`.

## Conventions

- **ID**: `<phase>.<n>`. **Status**: вҳҗ В· в—җ В· вҳ‘ В· вҠҳ deferred. **Size**: S/M/L.
- **Source of truth**: the kit is frozen (design); v1 product rules are the
  **Confirmed decisions** above (this doc is their in-repo home). Never edit the
  kit to fit Flutter. If a rule is missing, STOP and confirm вҖ” don't invent.
- **Codegen**: Riverpod + Drift use `build_runner`. Generated `*.g.dart` /
  `*.drift.dart` are build outputs; a CI gate checks they're up to date.
- **Strings** from ARB, never hardcoded.

### Definition of Done (per task; track-specific tests)

1. **Built** at the mapped path, respecting layer contracts.
2. **Analyzes** вҖ” `dart analyze lib test` вҶ’ 0 issues; codegen up to date.
3. **Tested** вҖ” the right pyramid level: **domain** = pure unit tests
   (deterministic, no Flutter) В· **data** = Drift integration tests (in-memory DB)
   В· **primitives/composites** = widget + golden (light+dark) В· **screens** =
   widget tests over provider states + golden vs `shots/*.png`.
4. **Parity / correctness** вҖ” UI matches the kit for every state; domain matches
   the v1 product rules (Confirmed decisions, above) with edge cases covered.
5. **Ledger** вҖ” row(s) in В§Ledger.
6. **Gates green** вҖ” `gen_tokens --check` + `dart analyze` + `flutter test` +
   codegen check.

### Verify

```bash
dart run build_runner build --delete-conflicting-outputs   # riverpod + drift codegen
node tool/design/gen_tokens.mjs --check
dart analyze lib test
flutter test
```

## Risks

- **R1 вҖ” Golden brittleness** across fonts/platforms вҶ’ load kit font in tests;
  canonical host; tolerances.
- **R2 вҖ” SRS correctness** вҖ” the 8-box scheduler is the product; a bug is silent
  data damage вҶ’ exhaustive unit tests before any screen consumes it (DM.4 gates S).
- **R3 вҖ” Drift migrations** вҖ” schema drift corrupts user data вҶ’ migration tests
  from v1 forward, never edit a shipped schema in place.
- **R4 вҖ” FE/BE contract drift** вҖ” if DM interfaces change after screens code
  against them вҶ’ treat DM.3 as a frozen contract; version changes explicitly.
- **R5 вҖ” Feature-local scope creep** (65 local components) вҶ’ pilot dashboard,
  size-check each screen.
- **R6 вҖ” Undrivable kit states** (error/loading) вҶ’ document as gaps, don't fake
  [[kit-to-flutter-error-state-unreachable]].
- **R7 вҖ” Kit drift mid-build** вҶ’ `gen_tokens --check` surfaces it; regenerate.
